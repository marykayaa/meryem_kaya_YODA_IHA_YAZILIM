Yazılım geliştirme süreçlerinde versiyon kontrol sistemi olan Git ve uzak depo hizmeti sağlayan GitHub'ın temel kavramları aşağıda açıklanmıştır:

### 1. Repository (Repo / Depo)

* Bir projenin tüm dosyalarını, klasörlerini ve o projenin başlangıcından itibaren gerçekleşen tüm değişiklik geçmişini (commit'leri) barındıran ana çalışma alanıdır.
* Proje versiyon takibinin yapıldığı ana kaynaktır.
* Bir projeyi GitHub'dan indirmek (klonlamak) için:
    ```bash
    git clone [GitHub Repo Adresi]
    ```

### 2. Commit (Kayıt / Değişiklik Seti)

* Projede yapılan bir dizi değişikliğin, anlamlı bir mesajla birlikte Git veritabanına kalıcı olarak kaydedilmesidir. Bir commit, projenin o andaki fotoğrafını çeker.
*  Yapılan işi küçük, anlaşılır ve geri dönülebilir parçalara ayırmak.
*  Hazırlanan dosyaları kaydetmek için:
    ```bash
    git commit -m "Odev 2 dokumantasyonu eklendi"
    ```

### 3. Branch (Dal)

*  Projenin ana kod hattından (genellikle `main`) ayrılan ve bağımsız geliştirme yapmaya olanak tanıyan paralel bir çalışma hattıdır.
* Ana kodun bozulmasını engellemek için yeni özellikler ayrı bir dalda geliştirilir.
*  Yeni bir dal oluşturup o dala geçmek için:
    ```bash
    git checkout -b odev_2
    ```

### 4. Stage (Hazırlama Alanı / Staging Area)

* Çalışma dizinindeki değişiklikler ile bir sonraki `commit` işlemine dahil edilecek değişiklikler arasında bir ara bölgedir. Hangi dosyaların kaydedileceğini seçmeyi sağlar.
* Çalışma dizinindeki tüm değişiklikler arasından, sadece istenen dosyaları kayda (commit'e) dahil etmek.
* Değişiklikleri Hazırlama Alanı'na eklemek için:
    ```bash
    git add .
    ```

### 5. Merge (Birleştirme)

*  İki farklı geliştirme dalındaki değişiklikleri bir araya getirme işlemidir. Genellikle bir özellik dalındaki çalışma tamamlandığında, ana dala aktarılır.
*  Bağımsız bir dalda tamamlanan işi ana projeye dahil etmek.
* `odev_2` dalını `main` dala birleştirmek için (önce `main` dala geçilir):
    ```bash
    git checkout main
    git merge odev_2
    ```

### 6. Pull (Çekme)

*  Uzak depodan (GitHub) hem en son commit'leri indirme hem de bu değişiklikleri yerel çalışma alanına uygulama işlemidir. Esasında `git fetch` ve `git merge` işlemlerinin birleşimidir.
*  Yerel repoyu ekip arkadaşlarının son çalışmalarıyla güncel tutmak.
*  Uzak depodaki `main` dalındaki güncellemeleri çekmek için:
    ```bash
    git pull origin main
    ```

### 7. Push (Gönderme / İtme)

*  Yerel repoda kaydettiğimiz (commit ettiğimiz) değişiklikleri, başkalarının da görebilmesi ve erişebilmesi için uzak depoya (GitHub) aktarma işlemidir.
* Yerel çalışmayı yedeklemek ve ekiple paylaşmak.
*  Yereldeki commit'leri GitHub'a göndermek için:
    ```bash
    git push origin main
    ```
